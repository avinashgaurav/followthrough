2. Product Briefings Delivered
XYZ Day (Quick Overview)
   1.​ Supports AWS, GCP, Azure, and OCI.
   2.​ Kubernetes cluster spin-up in ~20-30 minutes after connecting a cloud account.
   3.​ Applications = namespaces (logical separations).
   4.​ GitHub / GitLab / local machine CI/CD workflow auto-generated; copy-paste into repo to activate
       deployment automation.
   5.​ Data store attachment in ~5 minutes.
   6.​ Observability: logs, traces, metrics.
XYZ (Main Focus)
    1.​ Permissions: read-only plus lifecycle permissions to switch resources on/off.
    2.​ Unified resource listing with cloud / type / region icons.
    3.​ Toggle supports 20-60% savings for non-prod resources when well-scheduled.
    4.​ Weekly scheduler with time-zone selection, capsule-based hour selection, drag-selection,
        estimated savings display.
    5.​ Team assignment, cost / budget display, budget alerts via Google Chat, Teams, Slack (custom
        webhook supported).
    6.​ Admin-only Scheduler Override (example: up to 5 hours).
    7.​ Resource Groups with Auto Order + Custom Order (drag-and-drop) + Start Delay / Stop Delay.
    8.​ Teams with custom roles.
    9.​ Recommendations engine based on 7-14 day utilization: right-sizing, idle detection, orphan
        detection; remediation steps for each; auto-apply on roadmap.
    10.​Storage policies addressable (acknowledging that many large organizations know the policies but
        have not applied them).
    11.​Settings / integrations to route switch on / off and budget notifications.
    12.​Overview dashboard: total savings to date, sync every 15 minutes, resource / group / team
        counts with delta indicators, time-window savings, storage / network / database / compute split,
        scheduler success rate with reason attribution (internal vs. hyperscaler).
    13.​AI Tagging via metadata for untagged estates - particularly relevant for large organizations.

3. Technical Evaluation - Tanmay's Probing Questions
Differentiation vs. Hyperscaler-Native Tools
   1.​ AWS / Azure natively provide some alerting and recommendations (including email and
       WhatsApp notification).
   2.​ XYZ differentiation: broader coverage, particularly Kubernetes stateful sets and deployment
       sets cannot be "switched off" to zero compute via native tools (only pod-level autoscaling).
   3.​ XYZ's recommendation posture includes proactive machine-size reduction - hyperscalers
       historically avoid such recommendations.
   4.​ Roadmap direction: suggest optimal schedule windows based on observed idle periods.
Recommendation Data Cadence
    1.​ Current observation windows: 7 days and 14 days.
