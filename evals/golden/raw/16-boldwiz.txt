Budgets and Alerts

3. Client Questions - Governance and Onboarding
Team Management
  1.​ Admin user defines teams; XYZ itself never gets write access that would let it disrupt
      production.
  2.​ Only read-only plus metadata and lifecycle permissions are taken.
  3.​ Adding a member: enter the organization email, assign a role (e.g., Admin) and click Save - team
      and access are provisioned.
AWS Resource Coverage
    1.​ Claim made: XYZ is highly compatible with AWS (their earliest cloud supported).
    2.​ Coverage includes Kubernetes, EC2, databases, and other services; client to be sent a PDF
        enumerating supported resources across AWS, Azure and Oracle.
Cloud Integration Mechanics
    1.​ Integration via "Connect your cloud account" wizard; AWS-specific permission explained as
        read-only plus lifecycle permissions only.
    2.​ No server IPs, usernames, or passwords are required.
    3.​ XYZ reads metadata, not application or network data.
